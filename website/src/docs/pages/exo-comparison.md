# Mesh and exo

Mesh and exo both explore local distributed inference. Mesh is focused on model serving through an OpenAI-compatible endpoint, model catalog metadata, and layer packages that can be reviewed and reused.

Use Mesh when you want:

- One local OpenAI-compatible endpoint.
- A public or private mesh of serving machines.
- Catalog-backed model refs.
- Layer packages for large models.
- Agent and tool compatibility.

Use exo when its peer-to-peer workflow and supported model/runtime path better matches your environment.
